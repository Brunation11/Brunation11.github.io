Defining Concepts With Ruby Classes

October 13, 2015

class Dog
 def initialize(name, age, color)
  @name = name
  @age = age
  @color = color
 end

 def my_name_is
  @name
 end

 def my_age_is
  @age
 end

 def my_color_is
  @color
 end
end

Here we have a simple class that defines the concept of our pet dog and just like any dog ours will have a name, an age, and a color. In Ruby concepts are often defined through data structures known as classes. They are collections of all the attributes and methods their concepts have. Even our dog has been given a name, age, and color attribute.

We've created a very basic concept of our pet dog, but classes aren't simply defined once. Our pup has the potential to grow and if we choose to we can give it other attributes such as gender or size or even breed. Basically anything that would define a real pup can be given to our dog class.

class Dog
 def initialize(name, age, color)

While this may sound a little confusing, it's actually quite simple and it's also the basis for our dog. We first define a class Dog, we do so by first declaring class followed by the name of our class capitalized. We then can predefine some attributes for our pup and here we've chosen name, age, and color. Our first pup might look something like this.

our_first_pup = Dog.new("Fluffy", 1, "white")

Here we have a variable our_first_pup and we've set it to Dog.new What this does is create a new instance of our Dog class. As you can see we then passed in a name, a number for age, and our pups color. Seems simple enough right?

class Dog
 def initialize(name, age, color)
  @name = name
  @age = age
  @color = color
 end

We've seen how initialize can have some pre-set attributes for our pet but what if we wanted to use these attributes somewhere else. For example what if we wanted a way to print our dogs name or age or color. We would need to use instance variables. An instance variable is created just like a local variable but is prefixed with an @. The main difference between the two is that while a local variable is defined and used within one method, an instance variable can be used through out all the methods of a class wherever it may be needed.

def initialize(name, age, color)
 @name = name
 @age = age
 @color = color
end

def my_name_is
 @name
end

def my_age_is
 @age
end

def my_color_is
 @color
end

So we have our new pup and we've given it a name, age, and color. Now we want to be able to print this information out. Well we can do so by creating different methods or actions for our pup. We have three different methods here but as you can see they're all meant to return one of our attributes. Since these are three different methods and initialize is a method in itself we needed a way to get our pups name, age, and color into these new methods. We did so using instance variables. Take a look in our initialize method we created instance variables and assigned them our attributes. Note that it's good practice to keep the names the same in order to avoid any confusion when we get to some more advanced class structures.

our_first_pup = Dog.new("Fluffy", 1, "white")
puts our_first_pup.my_name_is
puts our_first_pup.my_age_is
puts our_first_pup.my_color_is

I was kind of worried about his shyness there for a second but he definitely pulled through. It's really that simple, we created a method and assigned some action to it. We then called this method on our pup by using #bark.

It may seem fairly simple now and if it does, well congratulation you now have a better understanding of Ruby classes. Almost everything in real life exists in a similar structure to our class. A dog has attributes and features and can perform certain actions. It can belong to a class dog, which in turn belongs to a class pet, which is an animal that exists as a an object in the universe just like an instance of our dog class exists as an object in a hierarchy of classes in Ruby.
