#map The last Method You'll Ever Use

So no joke, there I was, week-5 of DBC winding down and it seemed like it was only as week ago that we got started with Ruby. Of course that may just be because it was only one short week ago that we had our introduction to Ruby, a dynamic object oriented programming language.

Part of what gives this little language its charm though are the vast built in methods and what they can do for their respective classes. However not all classes are alike and inheritance tends to be limited to one parent class. It would be great to just be able to write code once and share it between unrelated classes, so great in fact that the it’s already possible thanks to special containers called modules. Modules are containers for methods and constants and allow these objects to be mixed in with other respective classes. Enumerable, Ruby’s most useful module does just that.

By now we’ve covered Arrays and Hashes, and how they can help us store information. They each have their own methods for storing, deleting, and iterating through data, and thanks to modules they have the ability to use Enumerable methods as well. Since Enumerable is a module, its methods are able to mix in with other classes meaning that Rubyists like you and I don’t have to write the same methods all the time for each class. There is a catch though; while classes can access Enumerable methods just by us telling them to include Enumerable, it requires that the class first have an #each method.

However you choose to stock up your Ruby arsenal, there is one Enumerable method that stands out above the rest. The classes we’ve covered such as Array and Hash have their own versions of #each that allow you to iterate through a container and perform some action on each object.

[ 1, 2, 3 ].each { | n | n + 1 }

Seems simple enough right? Well, it is. #each is a powerful method that allows us to iterate through an Array or Hash and perform an action to each object like in the example above.
