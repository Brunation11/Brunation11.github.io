The Last Method You'll Ever Want to Use

A brief intro into Ruby Enumerable methods

October 5, 2015

So no joke, there I was, week-5 of DBC winding down and it seemed like it was only a week ago that we got started with Ruby. Of course that may just be because it was only one short week ago that we had our introduction to Ruby.

Part of what gives this little language its charm though are the vast built in methods and what they can do for their respective classes. However not all classes are alike and inheritance tends to be limited to one parent class. It would be great to just be able to write code once and share it between unrelated classes, so great in fact that it’s already possible thanks to special containers called modules. Modules are containers for methods and constants and allow these objects to be mixed in with other respective classes. Enumerable, Ruby’s most useful module does just that.

By now we’ve covered Arrays and Hashes, and how they can help us store information. They each have their own methods for storing, deleting, and iterating through data, and thanks to modules they have the ability to use Enumerable methods as well. Since Enumerable is a module, its methods are able to mix in with other classes meaning that Rubyists like you and I don’t have to write the same methods all the time for each class. There is a catch though; while classes can access Enumerable methods just by us telling them to include Enumerable, it requires that the class first have an #each method. Basically we need the object in which we're calling an Enumerable method on to first belong to a class that supports interating.

However you choose to stock up your Ruby arsenal, there are some Enumerable methods that stand out above the rest, but first we need to understand how a more common iterator like #each works. The classes we’ve covered such as Array and Hash have their own versions of #each that allow you to iterate through a container and perform some action on each object.

my_array = [ 1, 2, 3 ]
my_array.each { | num | print "#{num}" }

my_hash = { "Paul" => "male", "Ana" => "female", "Mike" => "male" }
my_hash.each_pair { | name, gender | print "#{key} is a #{gender}" }

Seems simple enough right? Well, it is. #map is a powerful method that allows us to iterate through an Array or Hash and perform an action or simply print each object to the screen like in the example above.

While #each does a nice job of iterating, Enumerable gives us a few other methods with a bit more kick to them. Some of these include methods such as #map and #select.

#map returns a new Array with the results of running a block for each element. Think of it like e#each but with the added bonus of returning a new Array instead of modifying the original.

my_array = [ 1, 2, 3 ]
my_array.each { | num | num + 1 }

Our other super star Enumerable method is #select.This handy method returns an Array or Hash containing all elements where a given block returns true. This gives us the ability to select only content we specify to either return or be modified.

my_array = [ 1, 2, 3, 4, 5, 6 ]
my_array.select { | num | num.even? }
[ 2, 4, 6 ]

my_hash = { "Paul" => "male", "Ana" => "female", "Mike" => "male" }
my_hash.select { | name, gender | gender == "female" }
{"Ana" => "female"}

This gives us only a small taste of all we can do with Enumerable methods. The mixin offers colection classes such as Array and Hash methods for traversing,searching, and sorting elements. For more Enumerable methods take a look at Ruby-docs and try taking a few others for a test drive.
