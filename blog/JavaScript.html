Is a Hash Just an Object?

Ruby Hashes vs JavaScript Objects

October 22, 2015

You never realize how much you actually know until you stumble across something new. It’s week-7 of DBC and we just got our first taste of JavaScript. Having a good foundation in Ruby helped immensely with the logic. It’s like I said, you don’t realize how much you know until you stumble across something new, and JavaScript was definitely something new.

The great part about JavaScript and basically any programming language is that the logic isn’t all that different. In fact one of the biggest similarities in their fundamentals is how similar Ruby hashes are to JavaScript objects.

To put it simply, a Hash in Ruby is just a container that can store elements. Elements in a Hash aren’t stored in any particular order though in newer versions of Ruby hashes do remember the order that elements were added in. These elements are stored in Key Value pairs where by referencing the Key our Hash can return its Value pair.

state_abreviations = {
"New York" => "NY",
"New Jersey" => "NJ",
"Pennsylvania" => "PA",
"Virginia" => "VA",
"North Carolina" => "NC",
"South Carolina" => "SC"
}

puts state_abreviations["New York"]
:> NY

So we can see that a Hash is a container and that we can reference a Key and have it’s Value returned. If it seems simple that’s because it actually is. While the syntax might look a bit different an Object in JavaScript works just about the same. Objects in JavaScript can be created just like hashes in Ruby. We call this literal notation, meaning that we define a variable name and set it equal to a pair of curly brackets { } with our Key Value pairs nested inside. While in Ruby we simply refer to these pairs as Key Value pairs, JavaScript refers to them as properties. If we wanted to retrieve the Value from a Property in our object we would reference the Property just like referencing a Key in Ruby.

stateAbreviations = {
New York: "NY",
New Jersey: "NJ",
Pennsylvania: "PA",
Virginia: "VA",
North Carolina: "NC",
South Carolina: "SC",
};

console.log(state_abreviations["New York"]);
:> NY

One thing is for sure; there are many different programming languages each with their own unique syntax. Lucky for us the logic we pick up along the way can always be applied to learning something new. Ruby hashes and JavaScript objects are a great example and while they each have their differences, they are more alike than we think.
