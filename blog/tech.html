Regex Match and Replace

Intro to Regular expressions

November 01, 2015

As week-8 ends we’re one step closer to the end of phase-0 and the start of DBC’s onsite curriculum. We’ve covered numerous topics and even had our stints with Ruby and JavaScript and while nothing compares to learning the basics there are some more advanced concepts that we haven’t touched on.

In Computer science, pattern matching is the act of checking a given sequence for components of a specified pattern. Imagine using a method like each in Ruby and passing a block that checks if a variable matches a specific parameter. Some uses for pattern matching include returning the index of a match, the match itself, or to substitute the match also known as search and replace.

One such method of pattern matching is through the use of regular expressions; also know as regex or regexp. Regular expressions are sequences of characters that define search parameters for pattern matching. In the 1950’s Stephen Kleene, an American mathematician, defined what’s referred to as a regular language which is expressed through the use of strings or symbols and is constrained by the rules that govern it. It originally came into use with the Unix text processing utilities through the line editor ed, and the command line utility, grep.

While this may all sound very technical and somewhat of a hassle it is in fact worth the trouble. Many programming languages including the ones we’ve covered so far here during phase-0 of DBC have regular expression capabilities. While some languages have these built in like in Ruby and JavaScript, others such as Java and Python require a standard library for their regular expression capabilities.

The way pattern matching works in regular expressions is that each character is interpreted as a Meta character, a character with special meaning, or as a regular character with it’s literal meaning. In combination these characters make up a language base designed specifically to represent target patterns in a concise and flexible way.

Lets take a look at the basic concepts. The following operations can be used to construct most regular expressions. A Boolean can be used to match alternatives and is represented by a vertical bar. For example, gray|grey can match “their”, “there”, or “they’re”. More complex regular expressions are evaluated like basic arithmetic in a parenthesis is used to denote the precedence of operators. The pattern gr(a|e)y is the same as our Boolean example. A quantifier is another attribute of regular expressions and is used to specify how often a pattern check is allowed to occur. The three most common include ? , *, and +.

? indicates zero or one occurrences bike?r matches both “bike” and “biker.

* matches zero or more occurrences ab*c matches “ac”, “abc”, “abbc”, and so on.

+ matches one or more occurrences ab+c matches “abc”, “abbc”, “abbbc”, and so on.

Some other useful quantifiers include {n}, {min, }, and {min,max}.

{n} matches exactly n times.

{min,} matches min or more times.

{min,max} matches at least min but no more than max times.

Below is a basic table with some common Meta characters to get you started with regular expressions. Syntax does vary slightly between languages but we wont touch on that here, just keep in mind that regex has it’s own language base and the special meaning of each Meta character remains the same.

Meta Character Description
. Matches any single character
[] Match any literal character/s incased in them
[^ ] Match any single character not incased in them
^ Matches starting position within a string
$ Matches ending position within a string
( ) Defines a sub expression. Like in basic arithmetic, characters within parenthesis are evaluated first
* Matches the preceding character zero or more times
{n} Matches any single character n times
{n, } Matches any single character n or more times
{m, n} Matches any single character m times and no more than n times
... Denotes a set of character matches
| Seperates alternate possibilities
\w Matches any alphanumeric character
\W Matches any non-alphanumeric character
\s Matches a whitespace character
\S Matches anything but a whitespace
\d Matches a digit
.\D Matches a non-digit
\A Matches the beginning of a string
\Z Matches the end of a string
[^...] Matches every character except the ones in the brackets
